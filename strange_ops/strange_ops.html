Lab 3: Strange Operations

Question 1:What do these three functions do?

>>> bin(20)
'0b10100'
>>> oct(20)
0o24 
>>> hex(20)
0x14

These three functions take in some base 10 integer and return the same integer in the given base that is specified. For example, using the bin() function returns the base 10 integer input in binary, the same process occurs when using oct() and hex(), returning octal and hexadecimal numbers respectively.

Question 2: There is an infix binary operator ^, what does it do?

>>> 0b11 ^ 0b10
1
>>> 0b00 ^ 0b00
0
>>> 0b00 ^ 0b01
1
>>> 0b10 ^ 0b01
3

The infix binary operator ^ lines up each bit of two binary numbers. It then returns a new number in which bit combinations of 1 & 0 return 1, 0 & 0 return 0, and 1 & 1 return 0. It then takes the result of this comparison and returns the number which corresponds to the matching of each individual bit in the two number inputs.

Question 3: There is an infix binary operator |, what does it do?

>>> 0b11 | 0b10
3
>>> 0b00 | 0b00
0
>>> 0b00 | 0b01
1
>>> 0b10 | 0b01
3

The infix binary operator above lines up each bit of two binary numbers. It then compares bit by bit, if the bit pair is 1 and 0 one is returned, if the pair is 1 and 1 one is returned, and if the pair is 0 and 0 zero is returned. After returning the output of the bit comparison the new number is the result.

Question 3: There is an infix binary operator &, what does it do?

>>> 0b11 & 0b10
2
>>> 0b00 & 0b00
0
>>> 0b00 & 0b01
0
>>> 0b10 & 0b01
0

The infix binary operator above lines up each bit of two binary numbers. It then compares bit by bit, if the bit pair is 1 and 1 one is returned, otherwise 0 is returned. After returning the output of the bit comparison the new number is the result.
